📤 导出格式详解

打完轴，导出成你需要的格式。SUG 支持 11 种主流卡拉 OK / 字幕格式。

🚪 如何导出

  1. 切到「导出」界面侧边栏点击「导出」。
  2. 选择格式从下方 11 种格式里挑一个。
  3. 配置选项如演唱者过滤、是否插入【演唱者名】标签等（视格式而定）。
  4. 导出文件选择保存位置完成导出。
  5. 下次一键快导之后按 Ctrl+E 即可用上次的格式与目录快速导出。

💡 全局偏移已内置

所有导出器都读取已应用全局偏移后的时间戳，预览看到的就是导出得到的，不存在「导出后整体偏一截」的问题。

🎤 演唱者相关导出选项

导出时可按演唱者过滤（只列当前实际使用的演唱者）。详见 演唱者管理 → 导出时的演唱者处理。

🏷️ Nicokara：标签设置 + 演唱者名 + 分色助手

当导出格式选为 Nicokara / Nicokara（带注音） 时，会多出三个相关入口。它们是配套使用的， 搞懂彼此关系就能做出多人分色的卡拉 OK。

① Nicokara 标签设置…

设置文件头部的元数据标签：@Title（歌曲名）、@Artist（演唱者）、 @Album（专辑）、@TaggingBy（打轴者）、@SilencemSec（前奏静音毫秒）、 以及任意 @Custom 自定义标签。保存后写入软件配置，导出时写进文件头。

② 插入【演唱者名】标签

勾选后，导出时每当演唱者发生变化，就在该处字符前自动插入一个 【演唱者名】 标签。子选项「→每行行首都插入演唱者」可让每行开头都带上演唱者名 （需先勾选上一项）。这个 【名】 标签就是分色的「触发字符」。

③ 分色标签设置助手…

这是把「演唱者」变成「颜色 / 图标」的关键。它按演唱者逐行配置 NicokaraMaker3 的 @Emoji 标签，确认后自动写入 Nicokara 标签的自定义字段（即 @Custom）。格式：

@Emoji=<触发字符>,<擦除前图片>,<擦除后图片(可省)>,<选项…>
  • 触发字符：通常就填 【演唱者名】 —— 与第②步插入的标签对应。
  • 前/后画像：演唱者的图标图片名（仅取文件名）；纯分色无图标时常用一张透明 1×1 占位图。
  • 选项：Zoom=n% 缩放、Fix 原图尺寸、NoDecor 不加文字装饰、 ForceWipeDecor 强制擦除装饰、MarginLeft/Right/Bottom（余白，可负）。
  • 演唱者列表跟随导出过滤器的勾选（没勾就用全部演唱者）；可「将首行参数复制到其他行」批量套用。
典型「无图标纯分色」写法：
@Emoji=【ハル】,透明画像1x1.png,,Zoom=1,NoDecor,MarginRight=-170
🔗 三者怎么联动起来

① 分色助手定义「每个 【演唱者名】 触发什么图标/装饰」并写进 @Custom； ② 插入【演唱者名】标签把这些 【名】 触发字符真正插进歌词的换人处； ③ 导出的 Nicokara 进 NicokaraMaker 后，文字里的 【名】 命中对应 @Emoji 规则， 就实现了按演唱者分色 / 加图标。三步缺一不可：只插标签没配助手 = 看到一堆 【名】 文字； 只配助手没插标签 = 没有触发点、不生效。

📥 导入兼容性（反向）

SUG 也能导入既有成果继续编辑：

  • 支持导入 ASS / LRC / TXT / SRT / KRA / Nicokara 以及 RL 编辑模式内联文本。
  • 创建项目时自动识别格式（含 LRC 的逐行/逐字/增强型三种子类）。
  • RhythmicaLyrics导出的文件可直接导入；联网词典也兼容 RL 官方接口，旧词典资产可复用。
