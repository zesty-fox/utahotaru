🇯🇵 自动注音详解

给日文歌词的汉字标假名（furigana / 读音），SUG 提供多引擎 + 词典 + LLM + 罗马音的完整方案。

什么是注音？为什么打轴需要它？

日文汉字一个字可能对应多个假名（拍 / mora）。注音告诉软件「大冒険」读作「だい・ぼう・けん」， 这样它才能正确地把字拆成节奏点、在卡拉 OK 里逐拍变色。注音准不准，直接影响节奏点切分。

🤖 注音引擎与回退链

SUG 按不同发行变体使用不同主引擎，再叠加词典与回退规则：

变体主注音引擎
main（Windows 主版本）WinRT JapanesePhoneticAnalyzer（系统自带日语 IME 分析器）
noWinIME / macSudachiPy（内置，无需系统语言包）

主引擎之外，注音结果会经过一条回退链逐级兜底：用户词典 → e2k 英文引擎 → 用户词典整词回退 → e2k 静态词表 → 英文 fallback；日语侧另有 pykakasi + 本地/联网词典作参考与覆盖。

💡 词典优先级最高

本地 / 联网词典的读音会覆盖引擎结果。遇到生僻词、专有名词、歌曲特殊读法读不准时， 把正确读音加进用户词典即可一劳永逸。

🔤 按字符类型启停注音

注音可对不同字符类型分别开关 —— 比如只给汉字注音、保留假名原样。支持的类型包括：

平假名片假名汉字 字母（英文）数字符号 空格小假名 ぁぃぅぇぉゃゅょゎっ

软件还内置了一批规则，自动处理：句首句尾、促音（っ）、长音（ー / ～）、括号、英文音节切分等。 「按类型删除注音」（Ctrl+4）可按类型批量清除， 例如一键去掉所有符号 / 长音的注音。

🧩 连词（多字共享读音）

有些词需要多个汉字共享一段读音、作为一个整体变色（如「大冒険」连成一组）。这就是连词。

  • 用 F3 切换当前字与下一字是否连字。
  • 连词组在预览里会合并渲染读音与连词框。
  • 内联格式写作 {大冒険|だい,ぼう,けん}，各字独立读音用逗号分隔； {One|ワン,,} 表示首字承载读音、其余为空。

🔡 罗马音与「仅注音」模式

  • 一键转罗马音：把假名注音整体转成罗马字（romaji），适合做罗马音歌词。
  • 仅注音模式：只生成 / 显示注音而不影响其它，便于专注校对读音。

🧠 LLM 注音（可选 · 自配 API）

对引擎和词典都难搞定的复杂歌词，可启用 LLM 注音：用大语言模型推断读音。

  • 自带 API（bring-your-own）：在设置里填入你自己的 API 配置。
  • 自动回退：LLM 调用失败时自动退回常规引擎，不中断流程。
⚠️ LLM 注音是可选项

它需要你自备可用的 API 且会产生网络请求 / 费用，适合追求高准确率的进阶用户。 日常注音用内置引擎 + 词典通常已足够。

📚 本地用户词典

用户词典是你专属的读音覆盖表，独立存于 dictionary.json，重置配置不会清空它。

  • 首次启动自动从内置 1757 条 RL 词典种入。
  • 在「设置 → 读音词典」管理：条目按优先级排列（新增置顶），可上移/下移调整。
  • 支持导入 RL 字典，格式 原文⇥读音1,读音2…，其中 ＋ 代表连词。
  • 允许同一个词有多个读音并存；最新导入的条目优先级最高。

🌐 联网读音词典

内置 RhythmicaLyrics 官方源，可在线查询读音并与本地词典合并使用。详细的源配置、 自定义 URL、优先级排序与定时更新见 设置详解 → 网络。
