🎯 打轴功能详解

SUG 的核心。本章把打轴涉及的每一类操作拆开讲清楚。

🎹 基础打轴操作

操作键说明
打轴（记录当前时刻）Space给当前字打上时间戳，并自动前进到下一字
清除当前节奏点Backspace只清时间戳，不删字。删除后会智能回跳一小段便于复听
打到句尾P把当前行剩余字符一次性全部打上时间戳
句尾标记切换.切换行末「●」句尾标记（决定一句何时结束变色）
💡 删除即回放

清除节奏点时，光标会自动回跳 jump_before_ms 毫秒，方便你立刻重听、重打 —— 返工不用手动找位置。

⏯️ 播放、跳转与变速

操作键默认值
播放 / 暂停D音频与光标同步
停止（回行首）S—
后退 / 前进Z / X后退 2000 ms / 前进 3000 ms
减速 / 加速Q / W±5%，范围 0.2×–2.0×
音量 − / +Ctrl+W / Ctrl+Q±5%

变速不变调，慢放扒轴

SUG 采用 BASS-FX + pedalboard 双管线时间拉伸，最低能拉到 0.2×仍保持音高不失真 —— 慢放时咬字、气口、连唱依旧清晰。扒快歌、密集歌词强烈建议先慢放打粗轴，再原速校准。

🔧 节奏点（checkpoint）精修

切到编辑模式后，可调整每个字内部的节奏点数量 —— 用于长音、拖腔、一字多段变色。

操作键说明
增加 / 减少节奏点[ / ]给当前字添加或移除一个字内节奏点
切换字内节奏点Alt+← / Alt+→在一个字的多个节奏点间移动选中
时间戳微调 ±Alt+↑ / Alt+↓当前节奏点 ±10 ms（步长可在设置改）
^ 是什么？—— 演唱停顿符

当一个字的节奏点数量多于它的读音拍数时，多出来的节奏点没有对应假名，会用 ^ 占位， 表示延音 / 拖腔 / 停顿（这一拍换色但不是新音节）。它是有意义的占位符，不是错误。 详见 全文本编辑 → 演唱停顿符。

🎵 导唱符（前奏 / 间奏指示）

导唱符（如 ●）用于在歌词开始前、间奏处显示「即将开唱」的节拍提示点。

  • 插入导唱符：Ctrl+1（默认符号「●」）。
  • 插入导唱符对话框：在工具栏，可在选中字符前批量插入导唱字符，自动计算时间戳并设置连接关系。
  • 待办标记：对还没打导唱符的位置，可标注「此处需导唱符」提醒自己。

编辑管理里的「修改选中行」

打开行详情对话框，逐字编辑该行的文本、注音、节奏点数量、句尾标记和演唱者 —— 适合集中整理一整行。

时间戳工具（特殊工具详解）

这是扒轴返工最常用的一组，里面几个「特殊工具」单独说明：

工具作用
补全时间戳打轴全部完成后，自动找出还没有普通时间戳的字符（可按字符类型 + 排除规则筛选），用前后时间戳的平均值均分补上；可设「行首扣除 / 行尾增加」的边界偏移。
分离符号时间戳针对标点 / 符号批量处理：后补偿 = 把句尾符号的句尾时间戳提升为普通时间戳、并把句尾时间后移；前补偿 = 把符号的时间戳传给它后面的第一个文字、符号自身时间前移。用来让符号与文字的时间「分家」。
调整原始时间戳
（全部 / 按行 / 所选）		非模态窗口，点「应用」就把（全部 / 该行 / 所选）的原始时间戳整体平移一次，正=后移、负=前移；可反复叠加、可切回主界面试听后继续调。
批量删除时间戳删除所有 / 删除所有但保留行首 / 删除所选范围
⚠️「调整原始时间戳」与「全局偏移」不一样

调整原始时间戳是真的改写每个字记录的原始时间值（不可一键还原，靠撤销栈）； 全局偏移则是整体平移显示/导出而不动原始值。整条轴系统性偏，优先用全局偏移； 只想永久改某段的原始值，才用调整原始时间戳。

✏️ 改字、批量替换与候补字典

  • 修改字 / 行：J（修改所选字符）/ Ctrl+2（修改行）。 修改所选字符对话框可改文本、注音、节奏点数量，并能注册到用户词典，自动保持句尾标记与演唱者继承。
  • 批量替换：Ctrl+H，整篇批量替换文本（支持撤销）。
  • 查询候补字典：在「批量变更 / 修改所选字符 / 编辑注音(F2)」对话框中，点「查询候补字典」可列出 词典里与原文完全匹配的读音条目，双击应用后自动按该读音分段填充。

🔊 打轴按键音

可选 默认 / osu / 街机风 / 金属感 等按键音效，把打轴体验拉近音游手感。默认关闭， 在「设置 → 打轴 → 打轴按键音」中按需开启，并可调音量与风格。

↩️ 撤销 / 重做与自动保存

  • 撤销 / 重做：Ctrl+Z / Ctrl+Y，命令式深度撤销栈，支持几十步回退。
  • 自动保存：默认每 5 分钟把工程落盘到程序目录 .cache 下的临时 .sug.temp。 间隔可在「设置 → 自动保存」调（1–60 分钟）。
  • 闪退恢复：异常退出后再次启动，会提示从临时文件恢复现场。
⚠️ 注意：打轴会覆盖现有时间戳

在已打过的字上再按 Space 会直接覆盖原时间戳，且不提示时间倒退。 重打前若想保留旧值，请先确认或利用撤销栈回退。
