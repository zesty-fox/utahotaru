🎤 演唱者管理

对唱、合唱、分声部 —— 给不同演唱者上不同颜色，导出时区分显示。

什么场景需要它？

双人对唱、团体合唱、主唱/和声分离 —— 这些场景里不同人唱的歌词通常用不同颜色区分。 SUG 让你把每一句、甚至每一个字指定给某位演唱者，并自动按其配色渲染、导出。

👥 管理演唱者列表

在侧边栏「演唱者」界面管理。每位演唱者可设置：

  • 名称与颜色（支持自动配色、互补色生成）。
  • 分色：单个演唱者最多 5 种颜色（用于渐变 / 多色描边等效果）。
  • 分组：把多位演唱者归到一组管理。
  • 拖拽排序：调整演唱者在列表中的顺序。
  • 启用 / 禁用：临时屏蔽某演唱者。
  • 软件级预设：常用演唱者配置存为预设，跨项目复用（独立存于 singers.json， 重置配置不影响）。

🖍️ 把演唱者指定给歌词

SUG 支持逐字（per-char）级别的演唱者指定，也支持整行批量：

操作键 / 方式说明
应用演唱者K把当前选中（单字或划选范围）指定给某演唱者
按行设置演唱者Ctrl+5整行指定演唱者
跨行划选批量应用鼠标框选 + K跨多行选中一段，批量指定同一演唱者
右键菜单设置右键字符上下文菜单里直接「设置演唱者」
💡 创建项目时已自动铺底

创建项目时，默认演唱者会传播到句 / 字 / 注音三层，避免留空。Nicokara 导入则通过 singer_key 映射达到同样效果。所以你只需在「换人唱」的地方改演唱者即可，其余自动继承。

📤 导出时的演唱者处理

导出界面提供两个独立的演唱者相关选项：

  • 按演唱者过滤：只导出某位/某些演唱者的歌词（导出界面会自动只列出当前实际用到的演唱者， 列表可滚动）。适合单独导出某声部。
  • 插入【演唱者名】标签（仅 Nicokara）：在演唱者切换处自动插入 【名】 标签。这是独立设置项，与过滤互不影响。

导出 Nicokara 时还有 Nicokara 标签设置 与 分色标签设置助手 —— 后者用 @Emoji 标签把每位演唱者变成颜色 / 图标，与「插入【演唱者名】标签」配套实现多人分色。三者如何联动见 导出 → Nicokara：标签设置 + 演唱者名 + 分色助手。
