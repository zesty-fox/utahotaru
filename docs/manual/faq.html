❓ FAQ / 排错

常见疑问与故障排查。找不到答案？欢迎到 GitHub Issues 反馈。

💬 常见问题

Q1. 为什么叫 “StrangeUtaGame”？这是音游吗？

不是。Uta 是日语「歌」。它是卡拉 OK 打轴工具，名字只是有点意思而已，不是音游。

Q2. 和 RhythmicaLyrics、Aegisub 有什么区别？

  • 相比 RhythmicaLyrics：现代 UI、毫秒级低延迟回放、无需日语转区、原生 UTF-8。
  • 相比 Aegisub kara-templater：SUG 专注「打轴」这一步，导出 ASS 后再交给 Aegisub 做样式与特效，两者互补。

Q3. 支持 macOS / Linux 吗？

官方主要支持 Windows 10/11；macOS 为实验性构建。代码是 Python + Qt，理论可移植， 欢迎社区贡献。

Q4. 我之前的 RhythmicaLyrics工程能导入吗？

能。支持导入 RL 导出的 ASS / LRC / TXT 与 RL 编辑模式文本；联网词典兼容 RL 官方接口， 旧词典资产可直接复用。

🛠️ 故障排查

🔇 打开后没有声音 / 某个音频加载报错

  • 发行版自带全套 BASS 解码 DLL（FLAC / OPUS / AAC / AC3 / WMA / DSD / ALAC / WebM）。 从源码运行时，确保 src/strange_uta_game/lib/ 下插件完整。
  • 视频文件靠内置转换器抽音轨；若失败，检查「设置 → 关于/语言 → FFmpeg 路径」， 可填路径或「一键安装」。
  • 检查系统输出设备与软件音量（Ctrl+Q 加音量）。

🈁 自动注音不准 / 某些词读音错

  • 主版本注音引擎是 Windows 的 WinRT JapanesePhoneticAnalyzer，需安装日语基本包 （Language.Basic~~~ja-JP）。未启用时软件会探测并引导安装。
  • 生僻词 / 专有名词 / 特殊读法：把正确读音加进用户词典，它会覆盖引擎结果（一劳永逸）。
  • 仍不满意可尝试联网词典或可选的 LLM 注音。详见 注音功能详解。

🌐 联网词典查不到 / 被墙

到「设置 → 网络」切换词典源（内置 RhythmicaLyrics 官方）、添加自定义 URL、调整源优先级。

🚀 自动更新下载很慢 / 失败

到「设置 → 网络」的更新区，启用 ghproxy / gh-proxy / ghproxy-net 镜像，或配置系统/手动代理； 也可「跳过此版本」。macOS 版无应用内更新，会用浏览器打开 Releases 页面。

💥 软件闪退后工程丢了？

别慌。SUG 默认每 5 分钟自动保存到 .cache 临时文件，下次启动会提示恢复。 平时也建议常按 Ctrl+S。

🈚 文件名 / 路径乱码？

SUG 原生 UTF-8，无需 AppLocale、无需日语转区。若你仍开着转区工具反而可能出问题，建议关闭。
