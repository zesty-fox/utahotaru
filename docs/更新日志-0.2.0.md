# StrangeUtaGame 更新日志：0.1.0 → 0.3.0

面向用户的版本更新说明。若需完整技术细节请翻阅 git log 与 `docs/fixes_summary.md`。

---

## ⚠️ 升级前必读

1. **项目文件格式升级**：.sug 工程文件由 v0.1 升级至 **v0.3.0**，Ruby 结构重构为 RubyPart 列表（方案 B）。升级后保存的工程**无法**被 0.1.0 版本打开，建议先备份。
2. **Ruby 数据结构变更**：旧的 `Ruby(text="わたし")` 字符串式注音已重构为 `Ruby(parts=[RubyPart, ...])`，每个 `RubyPart` 与父 `Character` 的一个 checkpoint 一一对应。**大部分用户无感**。「全文本编辑」界面注音语法为 `{原文||读音组1,读音组2,...}`，详见下文。
3. **主题支持**：支持浅色、深色、跟随系统三种主题模式，基于 qfluentwidgets 主题框架。
4. **快捷键配置会被迁移**：本版本引入「编辑模式 / 打轴模式」双模式快捷键，升级后旧配置会被自动迁移为双模式同键位，首次进入设置界面建议检查一次。
5. **Phase Vocoder 音频引擎已下线**：变速实现改用 pedalboard `time_stretch` 离线预渲染 + SPSC RingBuffer 架构，若你的工作流对旧变速音色有特殊依赖，请留意差异（下文「音频引擎」详述）。

## 🆕 新功能

### 编辑体验
- **全文本编辑界面**：最左侧新增行号列，可快速定位；导入歌词时的空行会被保留，导出时也不再被自动补入额外空行。
- **行号显示**：打轴界面、全文本编辑界面均新增固定行号列（不可编辑）。
- **双模式打轴**：新增 **打轴模式 / 编辑模式** 区分。音乐播放时自动进入打轴模式，暂停时进入编辑模式，左下角实时提示当前模式。
  - 同一按键在不同模式下可以绑定不同功能（如 Space 在打轴模式为「打轴」，编辑模式为「添加当前字符节奏点」）。
  - 设置界面按【描述名称】+【模式方案】组织，仅打轴模式 / 仅编辑模式 / 双模式分别用不同字体颜色区分。
- **字符时间戳微调**：新增 `Alt+↑` / `Alt+↓` 快捷键，对当前选中的节奏点做时间戳微调。步长可在「设置 → 打轴设定 → 微调时间戳步长」中修改（默认 10ms）。
- **字符内切换节奏点**：单个字符内有多个节奏点时，可在字符内循环选中下一个节奏点（默认 `Alt+→`）。
- **选中节奏点高亮**：被选中的节奏点会用「对应演唱者颜色的补色」渲染 `▮▶` 提示，明确当前正在操纵的目标。
- **批量字符编辑器**：重建为字符级行 + 逗号分隔 RubyPart，体现 ruby 分组；搜索项会自动预填第一个匹配的节奏点既有值。
- **右键菜单**（打轴 / 编辑模式共用）：新增 `删除选中字符 / 在后方插入空格 / 合并至上一行 / 删除本行 / 插入空行 / 添加节奏点 / 删除节奏点 / 设置切换句尾 / 设置演唱者`。划词情况下会成组处理。

### 注音（Auto Check）
- **英文 e2k 注音**：基于 CMU 字典 + morikatron/snippet 的 english_to_kana 规则引擎（已翻译注释为中文并在源码中声明来源）。
  - 注音优先级：**英文 e2k → 用户自定义词典 → pykakasi 库**。参与前一轮的字符不再进入下一轮。
  - 自动注音对象限定为：英文字母、英文词组、汉字、日汉字、平假名、片假名、阿拉伯数字。其他（如日语括号）不再误被注音。
  - 英文词组节奏点规则：**首字母 = 1 cp、中间字母 = 0 cp、末字母标句尾**。
  - Apostrophe 词边界：`what's` / `U.S.A.` 等作为整词识别，不再被切成 `what + ' + s`。
- **标点节奏点开关**：`()【】[]{}「」!?、` 等符号默认不生成节奏点（设置可开启）。
- **送り仮名 / 劣质拆分 fallback**：字典匹配到连词末尾有送り仮名时自动剥离；字典拆分质量过差时仅尾部 part 对应汉字会 fallback 到库注音。
- **连词判定**：放宽为同 `block_id` 相邻即认定连词；连续汉字间也会自动连词；用户词典拆分保留 `#` 隔离，不会被强制连词。

### Ruby 分组（v0.3.0 核心升级）
- **Ruby 重构为 RubyPart 列表**：`Ruby.parts = [RubyPart(text, offset_ms), ...]`，每个 `RubyPart` 对应父 `Character` 的一个 checkpoint。
- **「全文本编辑」界面注音语法**：`{原文||读音组1,读音组2,...}`
  - `||` 分开原文块与读音块
  - `,` 分开原文块内每个汉字的读音
  - `|` 分开同一汉字内多个 mora（即多个 RubyPart）
  - 例：`{大冒険||だ|い,ぼ|う,け|ん}` = 大(だ·い) 冒(ぼ·う) 険(け·ん)
  - 例：`{漢||か|ん|じ}` = 单字 3 个节奏点
  - 例：`{詩||うた}` = 单字单段
- **Ruby-cp 对齐**：`split_ruby_for_checkpoints` 以 cp 数为主导，对齐不丢字；空 RubyPart 防护已修复。
- **英文词组**保留整词作为一段 RubyPart，不会被主动拆为逐字母。

### 音频
- **音频引擎彻底重构**：从 Phase Vocoder 迁移到 **pedalboard `time_stretch` 离线预渲染变速**，配合 SPSC RingBuffer 零分配回调，立体声相位一致，消除爆音与卡顿。
- **离线预渲染 PCM**：大改架构改为离线预渲染变速 PCM + 极简回调，变速下不再因实时 DSP 抖动导致音频异常。
  - 速度切换时播控栏实时显示后台 WSOLA 渲染进度。
  - 新增 SPSC RingBuffer 作为音频回调零分配缓冲。
- **打轴渲染**：重写 wipe 时间线为行级插值，暂停旁路缓存，就近预热。
- **快捷键冲突提示**：编辑完立即判断，冲突直接拒绝并恢复旧按键，不再要求点「保存设置」；模式内判冲，不再跨模式误报。
- **设置即保存**：快捷键修改后切换页面即保存（与其他设置项一致），不必手动点保存。

### 导出
- **Nicokara 分演唱者导出**：过滤后若整行无字符会自动跳过；**用户主动插入的空行**任意情况都保留。
- **默认文件名跟随音频**：加载音频后导出面板的默认文件名会自动同步。

### 其他
- **演唱者预设持久化**：预设不再每次启动丢失。
- **自动保存**：.autosave 文件名规范化，不再与 .sug 重名。
- **撤销 / 重做**：命令模型固化覆盖到字符级对话框的「向后连词」字段等新字段。

## 🔧 改进

- **域模型不变式**：`Character.check_count` 只能通过 `set_check_count` setter 修改，配套 `RubyMoraDegradeError` 守护 ruby 数据一致性；全部 9 个外部写入点已迁移至 setter。UI 层在会引起 ruby 丢失的操作前会弹窗确认。
- **TimingService 统一入口**：`on_key_changed` 成为时间戳写入的唯一入口，句尾 cp 仅在 key-up 事件写入，与普通 cp 解耦。
- **文档精简**：`fixes_summary.md` 压缩为一行一批次的表格，经验性教训抽离到 `lessons_learned.md`。
- **设计文档同步**：清除 Phase Vocoder 相关章节，合并重复内容，全面更新 Ruby → RubyPart 语义。

## 🐛 重要修复

### 编辑 / 渲染
- 选中节奏点大白框（#9）根治（架构性方案，第 16 批）。
- 选中字符后 5–6 行 / 6–7 行间出现大片空白的渲染错位。
- 打轴界面切换到全文本编辑再切回会丢失 RubyPart 切分。
- 快捷键无法使用 `,` / `.` / `/` / `;` / `'` / `[` / `]` / `、` / `-` / `=` 等键位。

### 自动注音
- Ruby 分组空 part（残留空 RubyPart 导致的解析异常）。
- 用户字典英文读音（如 `Ro,me,o`）被误拆分。
- 字典英文读音在 ruby 渲染中丢失。
- 空 ruby 的汉字 `cp` 必须为 0；单字假名 `cp` 至少 1。
- 连续汉字间 / 库路径注音的连词回退逻辑修复。
- `update_checkpoints_from_rubies` 不再擦除 `linked_to_next`。
- 英文 contraction（`what's`）词边界，不再遗留裸 `'s`。

### 导出 / 导入
- Nicokara 演唱者过滤导出大量空行。
- 导入空行被吞 / 导出被自动补空行的不对称。
- Nicokara Ruby 跨块对齐。

### 音频
- 打破加载音频重入卡 UI 的循环。
- 速度切换时 UI 不再因后台渲染阻塞。

### 快捷键 / 设置
- 快捷键冲突检测完全失效 → 即时检测并拒绝；编辑模式 / 打轴模式独立判冲。
- 快捷键必须点「保存设置」才生效 → 切页即保存。
- 冲突提示「与哪一按键冲突」明确到名称。
- 默认主题文案位置调整为主题说明后缀。

### 域模型
- `Character.timestamps` 与 `check_count` 不一致 BUG：`set_check_count` 收敛唯一写入点 + trim/merge 语义 + `push_to_ruby` 清残留。
